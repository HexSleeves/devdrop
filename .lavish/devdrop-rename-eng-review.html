Finding 1 P1 confidence 9/10

A blind os.Rename leaves dangling absolute paths in config.json

init.go:44-45 persists an absolute identity path into config:

  44  if cfg.AgeIdentityPath == "" {
  45      cfg.AgeIdentityPath = filepath.Join(home, "identity.txt")  // ~/.devdrop/identity.txt

SaveConfig writes that absolute path to ~/.devdrop/config.json. If migration just renames the directory to ~/.devspace, the file moves but config.json still says AgeIdentityPath: ~/.devdrop/identity.txt — which no longer exists. Secrets decryption breaks. The manifest remote default (workspace_sync.go:273, appHome()/remotes/default) has the same exposure if it was stored.

Fix: the migration must rewrite stored absolute paths whose prefix is the old home dir, OR the code should stop persisting absolute appHome-rooted paths and resolve them at read time. See Decision 4.

Finding 2 Architecture — where the migration runs

appHome() is the single choke point (verified paths.go:15-24). Every home-rooted path flows through it. The question is where the one-time migration fires.

A · One-time in PreRun

rec

Dedicated migrateLegacyHome() in the root command's PersistentPreRunE. appHome() stays a pure resolver returning .devspace.

+ Explicit over clever; no FS mutation in a getter

+ Runs once, logs what it did

− One new call site to wire in

B · Inside appHome()

Do the rename check inside appHome() before returning.

+ Zero new call sites

− A getter that mutates the filesystem and stats twice on every call — surprising, and a perf smell

C · Read-both fallback

Never move anything; if .devspace is absent, read from .devdrop.

+ No mutation, safest against interruption

− Data lingers in old dir forever; every path resolver grows a fallback branch (not DRY)

Failure modes

Cross-version workspace break

Machine A (new binary) renames <ws>/.devdrop → .devspace. Machine B still on the old binary looks for .devdrop and finds nothing. Silent. No test, no error. Drives Decision 3.

Dangling config paths

Finding 1. Secrets fail to decrypt because AgeIdentityPath points at the old dir. User-visible error but confusing. Fixed by Decision 4.

Interrupted migration

os.Rename is atomic on same filesystem, so partial state is unlikely — but if home and temp cross filesystems it isn't. Guard: same-FS rename only, else copy-then-verify.
