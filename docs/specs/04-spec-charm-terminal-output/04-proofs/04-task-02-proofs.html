Task 02 Proofs - Root polish: fang wrapper and charm-log diagnostics

Sequence 04 · Feature: charm-terminal-output · Task 02 · Phase 3

Task Summary

This task wrapped the CLI entrypoint (cmd/devspace/main.go) with fang.Execute, giving styled help/usage/error output, automatic --version support, and hidden man/completion subcommands in one call — while keeping the existing version subcommand as a plain-text alias reporting the same underlying version string. It also added a shared charm.land/log/v2 logger (internal/devspace/diagnostics.go) and routed the startup/refresh diagnostics of the three long-running surfaces (watch, mount, hosted serve) to stderr through it, rendering leveled logfmt output when stderr is not a TTY.

What This Task Proves

  • fang wraps the root command with zero changes to command construction (NewRootCommand is unchanged and still testable directly with plain cobra output — only main.go changed).
  • The version subcommand and fang's automatic --version report the same configured version value, satisfying the single-source-of-truth requirement.
  • fang's styled rendering path is exercised by an automated test (not just manual observation): forcing CLICOLOR_FORCE=1 against a buffer writer produces ANSI-styled help, while the default (no forcing) path stays byte-clean — this is the audit-remediation coverage for the previously flagged TTY-only rendering gap.
  • watch's diagnostics are leveled, timestamped, and render as logfmt when piped, confirmed by both a CLI transcript and an automated test.
  • Unknown-command errors exit non-zero with a styled error box instead of a raw Go error dump.

Evidence Summary

  • CLI transcripts: --version, version, piped watch --once diagnostics, and an unknown-command error, all captured from a real built binary in a sandboxed workspace.
  • Automated tests: five new/updated cases in devspace_test.go and watch_test.go, plus the full existing suite, all passing.
  • Quality gates: go vet, golangci-lint v2.12.2, and govulncheck all clean, including under the fusetest build tag (the MountWorkspace signature lost its now-unused out parameter as part of this task).
